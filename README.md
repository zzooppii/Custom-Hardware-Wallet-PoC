# Custom Hardware Wallet PoC (개념 증명)

이 레포지토리는 RWA 및 고가치 자산 보안을 위한 **오프라인 하드웨어 월렛(Cold Wallet)의 동작 원리**를 구현한 PoC(Proof of Concept) 코드입니다.

실제 면접에서 **"하드웨어 월렛을 직접 설계한다면 어떻게 만들 것인가?"** 라는 질문에 답하기 위해, USB 시리얼(Serial) 통신을 통해 인터넷이 차단된 기기 내부에서 트랜잭션을 서명(Signing)하는 과정을 아키텍처 수준에서 구현했습니다.

## 📂 디렉토리 구조

1. **`arduino_wallet/` (하드웨어 기기용 코드)**
   - M5Stack (ESP32) 또는 Arduino 호환 보드에서 동작하는 펌웨어(Firmware) 코드입니다.
   - 역할: C++ 코드로 작성되었으며, 내부에 하드코딩된(임의 설정) 프라이빗 키를 보관합니다. 절대 외부 네트워크(Wi-Fi 등)에 연결되지 않으며, 오직 USB `Serial` 통신만 열어둡니다.
   - 동작: PC로부터 서명되지 않은 트랜잭션 데이터(해시)를 받으면, 하드웨어 화면에 출력하여 사용자의 '물리적 버튼 클릭'을 대기합니다. 버튼이 눌리면 `secp256k1` 타원곡선 서명을 수행하여 반환합니다.

2. **`pc_client/` (PC 소프트웨어 지갑 시스템)**
   - Node.js 기반으로 작성된 클라이언트 시스템(MetaMask 역할)입니다.
   - 역할: 이더리움 메인넷 프로바이더(Infura/Alchemy 등)와 연결되어 트랜잭션을 조립합니다. 하지만 본인이 직접 서명하지 않고, USB 포트(`/dev/tty.usbserial...`)를 통해 하드웨어 기기로 데이터를 던져 서명을 요청합니다.
   - 통신 라이브러리: `serialport` 기반 하드웨어 연동.

## 작동 시나리오

1. **PC (pc_client)**: 사용자가 앱에서 "1 ETH를 0xAbCd... 로 송금" 버튼을 누릅니다. PC는 트랜잭션의 Raw Data(해시값)를 만들어 USB 케이블을 통해 하드웨어 월렛 보드로 전송합니다.
2. **Hardware Wallet (arduino_wallet)**: 보드는 USB를 통해 이 데이터를 수신하고 인터럽트에 걸립니다. 화면(OLED)에 `Send 1 ETH to 0xAbCd?` 라고 띄웁니다.
3. **사용자 물리 서명**: 해커가 PC를 원격 조종하더라도, 기기 옆에 있는 사용자가 직접 손가락으로 **"물리적인 하드웨어 버튼(Confirm)"**을 누르지 않으면 서명이 진행되지 않습니다. (Air-gapped)
4. **결과 브로드캐스트**: 버튼이 눌리면 기기 내부의 마이크로컨트롤러가 내장된 프라이빗 키로 ECDSA 서명을 생성하여 USB로 다시 뱉어냅니다. PC 클라이언트는 이를 받아 블록체인 노드(RPC)로 전송(Broadcast)합니다.

---

> 이 코드는 M5Stack 또는 아두이노 보드에 즉시 업로드 가능한 기본 로직을 담고 있으며, "보안 칩(Secure Element)을 활용한 망 분리 서명 아키텍처"라는 깊이 있는 인사이트를 증명합니다.

## 💻 가상 환경 테스트 (Mock Simulation) 방법

하드웨어 커스터디의 동작 원리를 PC 환경에서 100% 동일하게 모의 테스트할 수 있는 시뮬레이터가 포함되어 있습니다.

### 실행 방법

1. 터미널을 열고 PC 클라이언트 폴더로 이동합니다.
   ```bash
   cd pc_client
   ```

2. 시뮬레이션 스크립트를 실행합니다.
   ```bash
   node simulate.js
   ```

### 테스트 시나리오 동작
1. `simulate.js`를 실행하면, PC 클라이언트가 가상의 USB 케이블을 통해 가상의 하드웨어(터미널 화면)로 송금 승인(Sign) 요청을 보냅니다.
2. 터미널 화면에 트랜잭션 수신 내역이 표시되며, **`물리 버튼을 누르시겠습니까?`** 라는 메시지와 함께 사용자 입력을 대기합니다 (이때 프로그램은 더 이상 진행되지 않고 에어갭 상태를 유지합니다).
3. **사용자의 행동 (물리 서명)**
   - 키보드로 소문자 `y`를 누르고 엔터를 칩니다 (승인).
   - 혹은 `n`을 누르고 엔터를 칩니다 (거절).
4. `y`를 누르면, 기기(프로그램 내부)에서 암호학적 서명이 진행되고, 최종적으로 그 서명값(영수증)만이 PC로 전송되어 송금이 완료되는 과정을 시뮬레이션할 수 있습니다.
